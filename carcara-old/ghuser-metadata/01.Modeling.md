## Decoded sources

### BuildingMeshes

**Source:** [`scripts/BuildingMeshes.py`](scripts/BuildingMeshes.py)

**Logic:** Iterates over each branch of the footprint DataTree, detects holes (courtyards) by testing polygon containment, and groups exterior polygons with their interior holes. For each building group, it generates a ground mesh and roof mesh as single-face ngons (using `Mesh.CreateFromClosedPolyline` for simple cases, or planar Brep meshing for polygons with holes), then builds lateral wall meshes by extruding each polygon edge upward by the given height. Empty branches are preserved in the output trees to maintain data structure consistency.

### IdentifyDuplicatePolylines

**Source:** [`scripts/IdentifyDuplicatePolylines.py`](scripts/IdentifyDuplicatePolylines.py)

**Logic:** Extracts vertices from each polyline, rounds their coordinates to the document tolerance, and computes a canonical signature that is invariant to both starting point and traversal direction (by rotating to the lexicographically smallest vertex and comparing both forward and reversed orderings). Groups polylines by identical signatures and outputs the indexes of all duplicates (excluding the first occurrence in each group) as a DataTree where each branch is one duplicate group.

---

### OffsetPython

**Source:** [`scripts/OffsetPython.py`](scripts/OffsetPython.py)

**Logic:** Processes each branch of the curve DataTree and maps offset distances to curves (single value applies to all; if count mismatches, cyclic modulo mapping is used). Each curve is tested for planarity before calling Rhino's `Curve.Offset()` with the specified corner style enum (None/Sharp/Round/Smooth/Chamfer). Failed offsets insert `None` placeholders so the output tree structure matches the input exactly.

---

### PointInsidePolygon

**Source:** [`scripts/PointInsidePolygon.py`](scripts/PointInsidePolygon.py)

**Logic:** Extracts polygon vertices and computes their geometric centroid (average of all vertex coordinates). Tests if the centroid lies inside the polygon using `point_in_polygon` from the legacy `carcara_geometry` module; if it does, the centroid is returned directly. If the centroid is outside (concave polygon), falls back to a polylabel iterative grid search that finds the pole of inaccessibility — the point farthest from all edges — using a `polylabel` function from the same module. The returned point is always at Z=0.

---

### SortByContainer

**Source:** [`scripts/SortByContainer.py`](scripts/SortByContainer.py)

**Logic:** For each container curve, extracts the curve's plane (falling back to WorldXY for non-planar inputs) and tests every input point for containment using `Curve.Contains()`. Point indexes that pass the containment test are added to the corresponding DataTree branch. The output always has the same number of branches as input curves — including empty branches for curves that contain no points — to ensure predictable tree structure for downstream `List Item` operations.

---

### ColorCalculator

**Source:** [`scripts/ColorCalculator.py`](scripts/ColorCalculator.py)

**Logic:** Maps a tree of numeric values to colors by interpolating across a user-supplied color gradient. The `cls` input controls classification: `0` produces a continuous gradient, a positive integer auto-bins values into that many classes (linear or percentile-based per the `lin` flag), and a list of floats defines explicit class boundaries. Outputs a color tree matching the input value structure, plus a legend mesh with per-vertex colors, text anchor points, label strings, and text heights — all sized and positioned relative to the `leg_pln` plane using configuration keys supplied via the `leg_cfg` text string.
