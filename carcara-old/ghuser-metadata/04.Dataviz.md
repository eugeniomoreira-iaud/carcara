# 04.Dataviz

A set of components for building data visualizations. They render custom graphics on the Rhino
screen and export them as fully-configured SVG for publication/editing. SVG export is folded in
here (as in the legacy plugin). `CurveDisplay` is a Rhino-viewport preview. Subcategory exposure
ordering: `1` primary → `4` tertiary.

## Decoded sources

Note on cluster format: `PolylineToSVG`, `CircleToSVG`, `NurbsToSVG`, `TextToSVG`,
`ScatterPlot`, `LinePlot`, and `Heatmap` are **GH clusters** (nested `ClusterDocument` inside
the archive). The SVG-converter clusters (`*ToSVG`) each embed a shared `PlanarMeshes`
sub-component for planar curve meshing; the chart clusters embed a `Save SVG` sub-component
(same source as the standalone `SaveSVG`). Chart math lives in the legacy module
`carcara-old/carcara/modules/carcara_charts.py`.

---

### CurveDisplay

**Source:** [`scripts/CurveDisplay.py`](scripts/CurveDisplay.py) (C# source stored with `.py` extension)

**Logic:** A C# `GH_ScriptInstance` that overrides `DrawViewportWires` to render curves directly in the Rhino viewport. On each `RunScript` call it parses an optional `Dash` string (space-separated dash and gap lengths) and subdivides the input curve into dash segments using arc-length parameterization. The resulting segments, colors, and widths are accumulated in per-instance lists and drawn with `args.Display.DrawCurve()`. `BeforeRunScript` clears the lists so each solution starts fresh.

---

### PolylineToSVG

**Source:** [`scripts/PolylineToSVG.py`](scripts/PolylineToSVG.py)  
**Cluster interface:** [`scripts/PolylineToSVG_interface.txt`](scripts/PolylineToSVG_interface.txt)  
**Shared sub-component:** [`scripts/PlanarMeshes.py`](scripts/PlanarMeshes.py) (embedded planar-mesh converter inside the cluster)

**Logic:** Delegates to `carcara_dataviz.extract_polyline_points()` to convert each Rhino polyline's vertices from Y-up to SVG Y-down coordinates using the canvas anchor and height. Closed polylines emit `<polygon>` elements; open polylines emit `<polyline>` elements. Stroke, fill, width, and dash pattern are resolved per-item (list or constant) via `get_indexed_value()`; `.NET Color` objects are converted to hex + separate opacity attributes for Adobe Illustrator compatibility.

---

### CircleToSVG

**Source:** [`scripts/CircleToSVG.py`](scripts/CircleToSVG.py)  
**Cluster interface:** [`scripts/CircletoSVG_interface.txt`](scripts/CircletoSVG_interface.txt)  
**Shared sub-component:** [`scripts/PlanarMeshes.py`](scripts/PlanarMeshes.py) (embedded planar-mesh converter inside the cluster)

**Logic:** Extracts center and radius from each Rhino `Circle` object via `carcara_dataviz.extract_circle_parameters()`, then transforms the center from Rhino Y-up to SVG Y-down coordinates using the canvas anchor point and height. Each circle becomes a `<circle cx cy r>` SVG element; stroke, fill, and opacity are resolved per-item with the same color-conversion helpers used by the other `*ToSVG` components.

---

### NurbsToSVG

**Source:** [`scripts/NurbsToSVG.py`](scripts/NurbsToSVG.py)  
**Cluster interface:** [`scripts/NurbsToSVG_interface.txt`](scripts/NurbsToSVG_interface.txt)  
**Shared sub-component:** [`scripts/PlanarMeshes.py`](scripts/PlanarMeshes.py) (embedded planar-mesh converter inside the cluster)

**Logic:** Approximates each NURBS curve as a polyline by sampling it at a configurable number of points (default 50) via `carcara_dataviz.extract_nurbs_path_data()`, which converts each sample point to SVG coordinates and builds an SVG `<path d="M … L …">` string. The sample count can be supplied per-curve for mixed-precision output. Stroke, fill, and opacity follow the same per-item resolution pattern as the other `*ToSVG` components.

---

### TextToSVG

**Source:** [`scripts/TextToSVG.py`](scripts/TextToSVG.py)  
**Cluster interface:** [`scripts/TextToSVG_interface.txt`](scripts/TextToSVG_interface.txt)

**Logic:** Accepts insertion data as either `Point3d` (no rotation) or `Plane` (rotation extracted from the plane's X-axis angle). Points are transformed to SVG coordinates via the canvas anchor; planes additionally yield a rotation angle passed to `svg_text_with_transform()`. A 1–9 justification integer is mapped to SVG `text-anchor` and `dominant-baseline` attribute pairs. Fill color and opacity are resolved via the shared color converter.

---

### Histogram

**Source:** [`scripts/Histogram.py`](scripts/Histogram.py)  
**Cluster interface:** [`scripts/Histogram_interface.txt`](scripts/Histogram_interface.txt)  
**Shared sub-component:** [`scripts/SaveSVG.py`](scripts/SaveSVG.py) (embedded inside the cluster for optional SVG export)

**Logic:** Computes bin edges and per-bin frequencies from the input value list, then scales bar heights to the canvas rectangle proportionally to the maximum frequency. Generates `Rectangle3d` objects for bars, `Line` objects for X/Y axes (with optional extension), and label anchor points at evenly spaced intervals along each axis. Horizontal grid lines at Y-label positions are generated when `gy` is `True`. The implementation is self-contained pure Python with no external chart-library dependency.

---

### ScatterPlot

**Source:** [`scripts/ScatterPlot.py`](scripts/ScatterPlot.py)  
**Cluster interface:** [`scripts/ScatterPlot_interface.txt`](scripts/ScatterPlot_interface.txt)  
**Shared sub-component:** [`scripts/SaveSVG.py`](scripts/SaveSVG.py) (embedded inside the cluster for optional SVG export)

**Logic:** A thin wrapper around `carcara_charts.create_scatterplot()`. Remaps X/Y data values onto the canvas rectangle and generates dot circles with configurable radii (constant or per-point list). Optionally maps a separate `col_vals` list (or the Y values if omitted) to a color gradient to produce a color legend alongside the chart. Returns axes, label anchors, grid lines, and — when `show_leg` is `True` — legend rectangles, colors, and label data from the same result dictionary.

---

### LinePlot

**Source:** [`scripts/LinePlot.py`](scripts/LinePlot.py)  
**Cluster interface:** [`scripts/LinePlot_interface.txt`](scripts/LinePlot_interface.txt)  
**Shared sub-component:** [`scripts/SaveSVG.py`](scripts/SaveSVG.py) (embedded inside the cluster for optional SVG export)

**Logic:** A thin wrapper around `carcara_charts.create_lineplot()`. Accepts X/Y data as flat lists (single series) or DataTrees (one series per branch) and delegates all chart geometry construction — line polylines, axes, label anchors, and optional X/Y grid lines — to the library function. Margin inputs (`mx`, `my`) shrink the effective data domain as a percentage of the data range before mapping to canvas coordinates.

---

### Heatmap

**Source:** [`scripts/Heatmap.py`](scripts/Heatmap.py)  
**Cluster interface:** [`scripts/Heatmap_interface.txt`](scripts/Heatmap_interface.txt)  
**Shared sub-component:** [`scripts/SaveSVG.py`](scripts/SaveSVG.py) (embedded inside the cluster for optional SVG export)

**Logic:** A thin wrapper around `carcara_charts.create_heatmap()`. Parses the input DataTree into a 2D matrix, interpolates each cell value across a user-supplied color gradient (minimum two colors), and generates a grid of `Rectangle3d` cells with matching color tuples. Produces row and column label anchor points, optional per-cell value labels, and a configurable legend bar (vertical or horizontal) with its own rectangles, colors, and label data.

---

### SaveSVG

**Source:** [`scripts/SaveSVG.py`](scripts/SaveSVG.py)

**Logic:** Combines one or more SVG element strings into a single body, derives canvas width and height from the canvas `Rectangle` geometry via `carcara_dataviz.canvas_origin_info()`, and wraps the body in a complete `<svg>` document with matching `width`, `height`, and `viewBox` attributes. Relative file paths are resolved against the Grasshopper document folder (falling back to the current working directory). The file is written only when `save_flag` is `True`; otherwise the assembled `svg_doc` string is still output for inspection.
