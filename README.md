# Carcara

Python-based Grasshopper plugin for **Rhino 8** that bridges PostGIS spatial
databases with parametric design workflows. Query, visualize, and export
geospatial data directly from the Grasshopper canvas.

> Full development spec and contribution rules live in [`CLAUDE.md`](CLAUDE.md).

## Repo layout

```
<repo>/                                 ← git repo root
├── carcara/                            ← THE deployable folder (copied as-is to UserObjects)
│   ├── crc_modules/                    ← the pure-Python package = the code (no Rhino imports)
│   │   ├── db/                         ← connection, query, spatial_query, writer
│   │   ├── geometry/                   ← wkt, offset, building_mesh
│   │   ├── rhino/                      ← Rhino-only: credentials dialog + sticky (excluded from pytest)
│   │   ├── svg/                        ← export, save
│   │   ├── utils/                      ← color, srid, sql_composer
│   │   └── viz/                        ← histogram, scatter, lineplot, heatmap
│   ├── userobjects/                    ← all built .ghuser files (committed; shipped to users)
│   └── version.txt                     ← generated by make_release.py; installer reads this
│
├── grasshopper/
│   ├── components/                     ← component SOURCE bundles (metadata.json, code.py, icon.png)
│   └── installer/                      ← two self-contained bootstrap installers (delivered in a .gh)
│       ├── install_python_libs.py      ← installs pip deps into Rhino 8 CPython
│       └── install_carcara.py          ← downloads repo, copies carcara/ → UserObjects
│
├── tests/                              ← pytest (imports crc_modules.*)
├── vendor/                             ← componentizer + GH_IO.dll (build toolchain)
├── build_userobjects.py               ← builds bundles → carcara/userobjects/*.ghuser
├── make_release.py                     ← build + write version.txt (one command before a release)
├── deploy.ps1                          ← dev convenience: copy carcara/ → UserObjects/carcara/
└── pyproject.toml
```

The source package lives *inside* the deployable folder (`carcara/crc_modules/`).
There is no separate `release/` copy — the committed `carcara/` folder **is**
what users download.

## Import name: `crc_modules`

Components import `from crc_modules.db.connection import ...`. Unique name → no
clash with a generic `import modules` elsewhere in the Rhino session.

## Deployed layout (after install)

```
%APPDATA%\Grasshopper\UserObjects\
└── carcara\                            ← the whole folder, copied by the installer
    ├── crc_modules\                    ← package; sys.path points at the PARENT (…\UserObjects\carcara)
    ├── userobjects\*.ghuser            ← Grasshopper scans UserObjects recursively, so these load
    └── version.txt
```

Each component's `code.py` puts `…\UserObjects\carcara` on `sys.path` (no
`__file__` in GHPython) then does `import crc_modules`.

## Install (end users)

Open **`grasshopper/installer/`** in your file browser and drag the `.gh` file
onto the Grasshopper canvas. It contains two `Boolean Toggle`-triggered
installers — run them once per machine in order:

1. **`install_python_libs.py`** — pip-installs runtime deps
   (`psycopg2`, `shapely`, `svgwrite`, `matplotlib`) into Rhino 8 CPython.
2. **`install_carcara.py`** — downloads the repo zip from
   `eugeniomoreira-iaud/carcara` (master branch) and copies
   `carcara/` → `UserObjects/carcara/`. Does a version check against the
   installed `version.txt`, keeps a timestamped backup, and rolls back on
   failure.

After step 2, Rhino's Grasshopper toolbar will show the **Carcara** ribbon with
32 built components across four subcategories (01.Modeling · 02.Queries ·
03.Utilities · 04.Dataviz). The built `.ghuser` files committed to
`carcara/userobjects/` are what the installer copies — no compilation step
needed on the user machine.

Both installers are **self-contained** (no `import crc_modules`) since they run
before the package exists.

## Develop

Full spec: [`CLAUDE.md`](CLAUDE.md) (architecture, component inventory, API contracts, rules).
Per-phase implementation plans: [`implementation/`](implementation/).

```powershell
pytest tests/ -v                              # run tests (import crc_modules)
python build_userobjects.py                   # bundles → carcara/userobjects/*.ghuser
python make_release.py                         # build + stamp version.txt
powershell -ExecutionPolicy Bypass -File .\deploy.ps1   # copy carcara/ → UserObjects/carcara/
```

## Requirements

| | |
|---|---|
| Rhino | 8 |
| Python | CPython 3.11+ (Rhino 8 ScriptEditor) |
| Database | PostgreSQL 14+ with PostGIS 3+ |
| DB driver | `psycopg2` |
| OS | Windows 10/11 (primary), macOS (secondary) |
