## Embedded Python Sub-component: GrasshopperGeometryToWKT

Found inside: `GeometriesWithSpatialFilter`, `ValuesWithSpatialFilter`, `CreateShapefile`.

Purpose: converts a GH geometry (point / line / polygon) to WKT string for injection into PostGIS SQL. Used at the cluster input to convert the `sp_ftl` spatial filter geometry and `geo` lists. Enforces uniform type across all branches and emits single or `MULTI*` WKT variants.

**Source:** [`scripts/GrasshopperGeometryToWKT.py`](scripts/GrasshopperGeometryToWKT.py)

**Source:** [`scripts/QuerySchemaNames_interface.txt`](scripts/QuerySchemaNames_interface.txt)

**Logic:** Runs a fixed `SELECT nspname FROM pg_catalog.pg_namespace WHERE nspname NOT LIKE 'pg_%'` through RunODBCQuery and returns the result list as `sch`. No SQLComposer substitution — the SQL is a constant with no user-supplied placeholders.

### QueryTableNames

**Source:** [`scripts/QueryTableNames_interface.txt`](scripts/QueryTableNames_interface.txt)
**Engines:** [`scripts/RunODBCQuery.py`](scripts/RunODBCQuery.py) · [`scripts/SQLComposer.py`](scripts/SQLComposer.py)

### GeometryEntities

**Source:** [`scripts/GeometryEntities_interface.txt`](scripts/GeometryEntities_interface.txt)
**Engines:** [`scripts/RunODBCQuery.py`](scripts/RunODBCQuery.py) · [`scripts/SQLComposer.py`](scripts/SQLComposer.py) · [`scripts/WKTtoGrasshopperGeometry.py`](scripts/WKTtoGrasshopperGeometry.py)

**Logic:** First detects the table's primary key via `pg_index / pg_attribute`; builds an `ORDER BY` clause from that PK. Then executes `SELECT ST_AsText(ST_Translate(ST_MakePoint("_cx", "_cy"), -_cx, -_cy)) FROM "SCHEMA"."TABLE" ORDER BY PK` — Cx/Cy are text values embedded verbatim into the SQL by SQLComposer, never float-parsed. Converts the WKT result list to GH geometry via WKTtoGrasshopperGeometry. A second query fetches PK values in the same order so geometry and keys remain parallel.

### GeometriesWithSpatialFilter

**Source:** [`scripts/GeometriesWithSpatialFilter_interface.txt`](scripts/GeometriesWithSpatialFilter_interface.txt)
**Engines:** [`scripts/RunODBCQuery.py`](scripts/RunODBCQuery.py) · [`scripts/SQLComposer.py`](scripts/SQLComposer.py) · [`scripts/GrasshopperGeometryToWKT.py`](scripts/GrasshopperGeometryToWKT.py) · [`scripts/WKTtoGrasshopperGeometry.py`](scripts/WKTtoGrasshopperGeometry.py)

**Logic:** The `sp_ftl` spatial filter geometry is first converted to WKT via the embedded GrasshopperGeometryToWKT sub-component. The cluster then detects the PK and runs a geometry SELECT with a WHERE clause of the form `ST_Intersects(ST_MakePoint("_cx","_cy"), ST_Translate(ST_GeomFromText(filterWKT, SRID), _cx, _cy))` — pushing the (already-local) filter boundary back to projected CRS via `+_cx, +_cy` inside SQL. The result WKT is converted to GH geometry; a parallel query retrieves PK values in the same order. The `f` input selects the filter function (0 = ST_Intersects, otherwise ST_Contains).

### ValuesWithSpatialFilter

**Source:** [`scripts/ValuesWithSpatialFilter_interface.txt`](scripts/ValuesWithSpatialFilter_interface.txt)
**Engines:** [`scripts/RunODBCQuery.py`](scripts/RunODBCQuery.py) · [`scripts/SQLComposer.py`](scripts/SQLComposer.py) · [`scripts/GrasshopperGeometryToWKT.py`](scripts/GrasshopperGeometryToWKT.py)

**Logic:** Converts `sp_ftl` to WKT via embedded GrasshopperGeometryToWKT, detects the table PK, then runs `SELECT COALESCE("TABLE"."COLUMN"::varchar, 'NULL') FROM "SCHEMA"."TABLE" WHERE <function>(ST_MakePoint("_cx","_cy"), ST_Translate(ST_GeomFromText(filterWKT, SRID), _cx, _cy)) ORDER BY "TABLE"."PK"`. The `f` input switches between ST_Intersects (0) and ST_Contains (other). NULL strings are replaced with `null_val`; results are grafted and returned as `v`.

### CreateShapefile

**Source:** [`scripts/CreateShapefile_interface.txt`](scripts/CreateShapefile_interface.txt)
**Engines:** [`scripts/RunODBCCommand.py`](scripts/RunODBCCommand.py) · [`scripts/SQLComposer.py`](scripts/SQLComposer.py) · [`scripts/GrasshopperGeometryToWKT.py`](scripts/GrasshopperGeometryToWKT.py)

**Logic:** Converts the `geo` geometry list to WKT via the embedded GrasshopperGeometryToWKT sub-component. Builds per-row value strings using SQLComposer, wrapping each geometry with `ST_Translate(ST_GeomFromText(wkt, SRID), _cX, _cY)` — adding the false origin back in SQL so projected coordinates are stored, never in Python. If `rt` is True, prepends a `DROP TABLE IF EXISTS` before `CREATE TABLE`. Executes the final DDL+INSERT via RunODBCCommand; `Fb` returns the execution feedback.

### CreateTable

**Source:** [`scripts/CreateTable_interface.txt`](scripts/CreateTable_interface.txt)
**Engines:** [`scripts/RunODBCCommand.py`](scripts/RunODBCCommand.py) · [`scripts/SQLComposer.py`](scripts/SQLComposer.py)

**Logic:** Zips column names (`c`) and types (`vt`) into column definition strings and formats row values (`v`) into SQL tuples. Builds a `CREATE TABLE ... (id SERIAL PRIMARY KEY, ...)` + `INSERT INTO ...` block using SQLComposer for `*SCHEMA*`, `*TABLE*`, `*COLUMNS_STR*`, `*COLUMNS_NAMES*`, `*VAL_STR*` substitutions (note: this component uses `*placeholder*` syntax, not `#placeholder#` — a pre-existing quirk of its SQLComposer template). Optionally prepends `DROP TABLE IF EXISTS` when `rt` is True. Executed via RunODBCCommand; `Fb` returns feedback.
